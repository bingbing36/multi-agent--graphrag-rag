# CODEX RULES

## Core Rules

1. 当修改或新增核心文件/核心逻辑时，自动提醒并建议同步更新 README 对应段落。
2. 遇到问题先给可选方案，让我选择后再实施，不先斩后奏。
3. 选定方案后，补全配套落地步骤（代码、运行流程、数据准备、验证步骤）。
4. 重要报错必须沉淀到 `ERROR_LOG.md`，保证可回溯。
5. 配置读取遵循当前约定：仅从 `.env.example` 读取（除非我明确改规则）。
6. 非必要不修改 `.env.example`（尤其是我已填写的个人密钥）。
7. 当任务效果不好时，主动推荐可用的 skills 或 harness，并说明适用场景与取舍。

## Working Rules

1. 每次修复后执行最小复测命令，并回传关键结果。
2. 报错提示要明确指出来源（DeepSeek / Qwen / Neo4j / Milvus 等）。
3. 涉及架构调整时，优先保证系统可运行，再逐步优化。

## Maintenance

- 本文件随协作偏好持续更新。
- 新规则默认追加到 `Core Rules` 并标注日期。
